# Contributing & Git Flow

- Work phase-by-phase on feature branches (phase-1, phase-2, etc.).
- Use conventional commit messages, e.g.:
  - `feat(phase-1): add scope summary and RACI`
  - `docs(phase-3): update encryption policy`
- Open Pull Requests to main and merge when reviewed.
- Tag releases at the end of each phase (v0.1-phase-1, …, v1.0-final).
